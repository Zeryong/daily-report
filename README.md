# git-report

一个 XXXCode Skill，自动扫描本地 Git 仓库的提交记录，生成结构化工作日报，并发布为飞书云文档。

## 功能概览

- 自动遍历指定目录下的所有 Git 仓库，提取指定日期的提交记录
- 按项目分组，统计提交次数、变更文件数、增删行数，推断涉及的模块
- 基于提交内容归纳"已完成"工作项，推测"进行中"和"明日计划"
- 调用 `lark-cli docs +create` 一键生成飞书云文档，返回文档链接

## 前置条件

| 依赖 | 说明 |
|------|------|
| Git | 本地已安装，且目标目录下有 Git 仓库 |
| lark-cli | 已完成 `lark-cli config init` 和 `lark-cli auth login`，具备云文档创建权限 |
| XXXCode | 本 Skill 在 XXXCode 会话中运行 |

## 使用方式

### 自然语言触发

在 XXXCode 对话中直接说：

```
写日报
生成日报
整理今日工作
写昨天的日报
帮我生成本周三的日报
```

Skill 会自动识别意图并执行。

### 手动调用

```
/daily-report
```

### 指定参数

- **指定日期**："写昨天的日报"、"生成 4 月 10 号的日报"
- **指定项目**："只看 xxx-xxx 项目的日报"
- **指定目录**："扫描 D:/work/projects/ 下的提交"

## 默认配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 扫描目录 | `c:/development/code/` | 遍历该目录下所有子目录寻找 Git 仓库 |
| 日期 | 当天 | 可通过自然语言指定其他日期 |
| 文档标题格式 | `工作日报 - YYYY年M月D日` | 自动填充日期 |

## 生成的日报结构

```
工作日报 - 2026年4月15日
├── 头部信息卡片（日期、提交人、项目数、提交次数）
├── 今日工作内容
│   └── [按项目分组]
│       ├── 一句话概括
│       ├── 提交详情表格（时间、内容、变更规模）
│       └── 涉及模块
└── 工作总结（三栏布局）
    ├── 已完成（基于提交记录）
    ├── 进行中（AI 推测，标注提示）
    └── 明日计划（AI 推测，标注提示）
```

## 文件结构

```
daily-report/
├── SKILL.md              # Skill 主文件（工作流定义 + 日报模板）
├── README.md             # 本说明文件
└── scripts/
    └── scan-commits.sh   # Git 提交扫描脚本
```

### scan-commits.sh

独立的 Bash 脚本，负责遍历目录并提取 Git 提交信息。

```bash
# 用法
bash scripts/scan-commits.sh [scan_directory]

# 示例
bash scripts/scan-commits.sh c:/development/code/
bash scripts/scan-commits.sh /home/user/projects/
```

输出格式为结构化文本，按项目分组，包含提交哈希、作者、时间、消息和变更统计。无提交时输出 `NO_COMMITS_TODAY`。

## 配合定时提醒

本 Skill 不包含定时调度功能，推荐配合飞书日历提醒使用：

1. 创建每个工作日 18:30 的重复日程提醒"提交日报"
2. 收到提醒后，在 XXXcode 中说"写日报"即可一键生成

## 注意事项

- "进行中"和"明日计划"是 AI 基于代码上下文的推测，生成后请按实际情况调整
- 当天无任何提交时不会创建文档，会直接告知用户
- 扫描仅覆盖指定目录的一级子目录，不递归深层嵌套仓库
- 需要飞书 lark-cli 的云文档创建权限（scope: `docx:document:create`）
